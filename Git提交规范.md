# Git 提交规范 (Commit Guidelines)

## 1. 提交格式 

每条 Commit Message 必须遵循以下格式：

```
[Type(Scope)] Subject
```

或者如果不涉及具体模块（Scope），可以简写为：

```
[Type] Subject
```

- **Type**: 提交的类别（如 Feat, Fix 等，见下表）。
- **Scope**: (可选) 修改的范围/模块（如 Player, Map, UI）。
- **Subject**: 简短的描述（建议使用中文）。

## 2. Type 类别

请严格使用以下关键字（首字母大写）：

| 关键字         | 含义       | 说明                                   |
| -------------- | ---------- | -------------------------------------- |
| **[Feat]**     | **新功能** | 新增了功能 (Feature)                   |
| **[Fix]**      | **修复**   | 修复了 Bug                             |
| **[Docs]**     | **文档**   | 仅修改了文档、注释或 README            |
| **[Style]**    | **格式**   | 代码格式调整（空格、缩进），不影响逻辑 |
| **[Refactor]** | **重构**   | 代码重构，既没新增功能也没修 Bug       |
| **[Perf]**     | **性能**   | 性能优化                               |
| **[Test]**     | **测试**   | 增加或修改测试代码                     |
| **[Chore]**    | **杂项**   | 构建过程、依赖库、资源文件(assets)变动 |

## 3. 示例

### ✅ 正确示例

- **新增功能（带范围）**：[Feat(Player)] 实现了角色移动和冲刺逻辑
- **新增功能（无范围）**：[Feat] 添加了背景音乐资源
- **修复 Bug**：[Fix(Enemy)] 修复了敌人死亡后贴图不消失的问题
- **文档修改**：[Docs] 增加了 MapGenerator 类的注释说明
- **代码重构**：[Refactor] 优化了碰撞检测算法
- **杂项（资源更新）**：[Chore] 更新了所有怪物的 png 图片素材

### ❌ 错误示例

- 增加了移动功能 (缺少 [Type])
- [Feat]：增加移动(方框后不要加冒号，虽然加了也基本没影响，但最好保持一致)
- update code (描述太模糊)
- [Bug] Fix error (请使用标准关键字 [Fix])

## 4. 建议

1. **保持原子性**：尽量一个 Commit 只做一件事。不要把“修 Bug”和“写新功能”混在一个 Commit 里提交。
2. **多提交**：不要写了一整天代码才提交一次，建议完成一个小功能点（比如写完一个函数，或者修完一个错）就提交一次。这样 Git 记录会非常丰富，**极大利于期末评分**。
