# Ark-knights
# 自选题目：元气骑士

**团队人数：** 2人

**游戏描述：** 游戏玩法参考《元气骑士》，设计一款 **2D 俯视角 Roguelike 游戏**。玩家将操控游戏角色在地牢中探索，击败敌人，挑战boss。

## 基础功能

### 1. 角色与操作系统

- **核心操作机制**：
  - 支持键盘控制角色移动。
  - 支持冲刺/闪避。
  - 支持普通攻击（平A）。
  - 支持释放角色技能，释放技能消耗法力值（MP）。
- **状态管理**：
  - 实现角色的生命值、法力值、攻击力等属性管理。
  - 支持基础状态切换（Idle/攻击/受击/死亡）。
- **角色实现（至少实现 1 名基础角色）**：
  - **法师**（远程攻击）
  - **炼金术士**（AOE伤害/Dot效果）
  - **重装战士**（近战攻击）

### 2. 战斗系统

- **敌人设计**：
  - **敌人种类**：
    - 支持至少3种基础敌人。
    - 支持至少1种boss敌人（有技能/有阶段变化）。
  - **敌人AI逻辑**：
    - 追击。
    - 攻击范围。
    - 部分敌人有特殊机制。
- **战斗判定**：
  - 实现碰撞检测。
  - 实现攻击范围判定。
  - 实现伤害数值计算与扣血逻辑。
  - 支持敌人死亡逻辑。
  - 支持boss血条显示。

### 3. 地图系统

- **地图机制**：
  - 采用预制房间拼接的方式，将不同房间连接，形成地图。
  - **锁门机制**：进入战斗房间后自动锁门，击败所有敌人后开门。
- **房间类型**：
  - 至少支持以下三种类型
    - **战斗房间**
    - **BOSS房间**
    - **奖励房间**
- **场景类型**：
  - 至少实现1种场景类型。
  - 不同的场景类型有不同的美术风格、背景音乐等。

### 4. 声音系统

- **音乐**：
  - 战斗背景音乐。
  - boss战特殊背景音乐。
  - 奖励音乐/通关音乐等特殊音乐。
- **音效**：
  - 不同的战斗音效（平A、释放技能、受击等）。
  - UI交互音效。

### 5. UI 与交互

- **HUD 界面**：
  - 显示玩家 HP 条、护盾值（如有）、MP/能量条。
  - 显示当前关卡层数。
  - 显示玩家技能冷却。
- **系统菜单**：
  - 开始界面：开始游戏、选择角色、退出游戏。
  - 暂停菜单（Esc键）：暂停游戏逻辑，支持继续或退出。
  - 死亡结算界面：显示“游戏结束”并支持重开。

## 可选功能 (可自由扩展)

- **战斗内容扩展**

  - **更多角色**：游侠、枪手、弓箭手等更多角色。

  - **更多怪物**：寄生怪物、亡语怪物、新boss等更多怪物。

  - **更多技能：**加入更多技能。

  - **天赋系统：**随着探索可以选择各种天赋，提升角色数值或完善角色机制。

  - **武器系统**：加入各种武器，玩家可以更换武器来提升战斗力。

- **收藏品扩展**

  - **收藏品收集**：设计 10+ 种具有不同效果的收藏品（如增加攻速/攻击力的道具）。

  - **套装效果**：实现集齐特定道具（如3件同系列藏品）触发额外强力 Buff 的逻辑。

  - **掉落机制**：击杀怪物或开启宝箱有概率掉落收藏品。

- **地图扩展**

  - **场景**：增加更多场景。

  - **关卡**：增加总关卡数。

  - **更多种类房间**：加入更多各种类型的房间，如彩蛋房间，商店房间（玩家击杀敌人获得金币，金币可以在商店内购买各种道具）。

  - **特殊地形**：加入各种特殊地形，例如陷阱、传送带、障碍物/掩体等。

- **可玩性扩展**

  - **难度分级**：可以选择简单（玩家获得buff）、普通、困难（敌人获得buff）三种难度进行游戏。

  - **存档/读档功能**：增加存档/读档功能，例如加入存档点房间，玩家可以在此房间存档或读取存档回到此房间。

  - **NPC系统**：加入各种NPC。

  - **经验系统**：完成探索可以积攒角色经验值提升角色等级，使角色拥有更强的战斗力。

  - **战斗评分系统**：完成单个房间战斗/通关后自动评分。
